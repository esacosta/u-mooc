CourseBuilder 1.1 le permite alojar múltiples cursos simultáneamente. Esto es, por supuesto, opcional. Si usted sólo tendrá que ejecutar un curso, deja de leer ahora. De lo contrario...

    • 1. Asegúrese de que los cursos se pueden ejecutar en la misma instancia.
    • 2. Crear o copiar archivos no compartidos en los directorios adecuados.
    • 3. Vamos Curso Constructor saber acerca de los múltiples cursos.
    • 4. (Sólo para aplicaciones de CB 1.0) Modificar las opciones de cada curso para utilizar URLs relativas.

1. Asegúrese de que los cursos se pueden ejecutar en la misma instancia.

Cuando aloja varios cursos en un caso, todos estos cursos deben compartir la mayor parte de su código. (Ver Explora el Código de ejemplo para obtener una descripción de todos los archivos enviados con desarrollador del curso.)

En concreto, se supone que AppPath es el directorio que contiene desarrollador del curso. Múltiples cursos que se ejecutan en una instancia GAE deben compartir todos los archivos y el código, excepto lo que está en el AppPath / activos, AppPath / datos, y AppPath /directorios de puntos de vista.

Si ya dispone de cursos que desee consolidar en una sola instancia, primero verifique que comparten el código correcto. Si este no es el caso, la parada aquí. No es posible acoger los cursos en la misma instancia.

2. Crear o copiar archivos no compartidos en los directorios adecuados.

En lugar de utilizar los activos de primer nivel, los datos y directorios de puntos de vista, cuando varios cursos comparten un ejemplo, utilizan un nuevo directorio de nivel superior, AppPath / cursos.

Este directorio contiene un subdirectorio independiente para cada curso. Cada uno de estos directorios se encuentran bienes, datos y directorios de puntos de vista para que un curso. Estos subdirectorios deben contener la totalidad del activo, los datos y ver los archivos para el curso dado.

Por ejemplo, suponga que tiene dos campos, álgebra y cálculo. Usted tiene archivos y directorios, tales como las siguientes:

AppPath / app.yaml 
... 
Apppath / cursos / 
Apppath / cursos / álgebra / 
Apppath / cursos / álgebra / activos / ... 
Apppath / cursos / álgebra / data / ... 
Apppath / cursos / álgebra / views / ... 
... 
AppPath / cursos / cálculo / 
AppPath / cursos / cálculo / activos / ... 
AppPath / cursos / cálculo / data / ... 
AppPath / cursos / cálculo / views / ... 
... 
AppPath / models / models.py 
...

Todos los archivos y directorios fuera de apppath / activos, apppath / datos, y apppath / vistas siguen siendo los mismos.

Si consolida los cursos existentes, realice una copia de un completo árbol AppPath. Mueva los activos de primer nivel, datos, puntos de vista y directorios en esa copia en un subdirectorio apropiado nombre para ese curso. A continuación, crear otros subdirectorios para sus cursos y copiar sus activos, datos y directorios de puntos de vista en el nuevo árbol.

3. Vamos Curso Constructor saber acerca de los múltiples cursos.

Añadir URL enrutamiento

Una vez que tenga los archivos en la estructura de directorios adecuada, es necesario configurar Course Builder para saber dónde se encuentran los cursos y qué direcciones URL a utilizar para referirse a ellos. Esto se hace mediante la variable de entorno GCB_COURSES_CONFIG en AppPath / app.yaml.

En nuestro ejemplo, supongamos que usted planea utilizar http://www.example.com/index.html como página de inicio que enumera todos los cursos, http://www.example.com/algebra101 para el curso de Álgebra, y http://www.example.com/calculus202 para el curso de Cálculo. En esta situación, debe agregar la siguiente sección para app.yaml.

  ...




 env_variables:




   GCB_COURSES_CONFIG: "Por supuesto :/ algebra101 :/ / cursos de álgebra, por supuesto :/ :/ / calculus202 cursos de cálculo '}}}




 ...

GCB_COURSES_CONFIG es una lista separada por comas de las normas. Cada regla tiene tres cuerpos, separados por dos puntos:

  • Tipo de entrada. 
    Cada regla debe comenzar con el     curso palabra. (Esto permite la expansión futura de las reglas de configuración.)
  • URL prefijo. 
    Especifica la URL a utilizar para esta parte del sitio. En el ejemplo, la página de inicio es accesible como     http://www.example.com/ y la segunda unidad del curso Algebra es accesible como http://www.example.com/algebra101/unit?unit=2.
  • Ubicación de archivo de sistema de archivos del curso. 
    Especifica dónde se encuentran los archivos asociados a cada curso. En el ejemplo, los archivos de vistas para Algebra 101 se almacenan en     apppath / cursos / álgebra / views /.

Espacios de nombres GAE

Al ejecutar varios cursos en una sola instancia GAE, Campo constructor necesita una manera de separar la información que pertenece a diferentes cursos. Por ejemplo, cada uno de los cursos tiene una unidad 1. Desarrollador del curso debe saber qué información corresponde a la Unidad 1 de Algebra 101 y que corresponde a la Unidad 1 de Cálculo 202. Para este propósito, se utiliza espacios de nombres GAE. Un espacio de nombres es un método para desambiguar colecciones de información.

Todos los cursos se ejecutan en una instancia GAE único, pero cada curso se ejecuta en espacio de nombres separado. Desarrollador del curso crea el nombre del espacio de nombres utilizando la ubicación del curso de sistema de archivo. Curso Constructor reemplaza las barras en el lugar con guiones y le da el espacio de nombres un prefijo de gcb. Así, en el ejemplo, el curso de Álgebra utiliza el nombre de espacio de nombres GCB-cursos-álgebra, en función de su ubicación en el sistema de archivos / cursos / álgebra.

4. (Sólo para aplicaciones de CB 1.0) Modificar las opiniones de cada curso para utilizar URLs relativas.

Si originalmente desarrollado sus cursos utilizando Course Builder 1.0, hay un paso final involucrados para ejecutar múltiples cursos de la misma instancia de aplicación. Este paso no se aplica si originalmente desarrollado los cursos de Curso Constructor 1.1.

En el Curso Builder 1.0, por defecto, muchos de los archivos de vistas / referencia a otros archivos del curso Builder utilizando las direcciones URL absolutas. Una URL absoluta comienza con http://, https:// o /. Por ejemplo, http://www.example.com/index.html y / index.html son URLs absolutas. Una URL que no comience por http://, https:// o / es una URL relativa, por ejemplo, index.html es una URL relativa. Una dirección URL absoluta describe precisamente donde vive la página, en "términos absolutos". Una dirección URL relativa se describen en los que la vida de la página, en relación con el directorio actual. Por ejemplo, en CB 1,0 visitas / base.html contenía la siguiente línea:

<link href="/assets/css/main.css" rel="stylesheet" type="text/css">

Al ejecutar el desarrollador del curso para una sola aplicación, / assets / css / main.css refiere a AppPath / assets / css / main.css.Al ejecutarlo para múltiples aplicaciones, lo que los activos / / css / main.css refiere a depender de la ruta definida porGCB_COURSES_CONFIG.

Para solucionar el URL, primero agregue el tag de base para vistas / views / base.html y base_registration.html. Agregue la siguiente línea al tanto de esos archivos, como la primera línea en la sección head.

<base href="{{ gcb_course_base }}" />

Esta etiqueta indica al navegador dónde empezar a buscar URLs relativas.

A continuación, compruebe todas las URL de tu curso Builder 1.0 cursos para asegurarse de que están bien.

  • Las URL absolutas que comienzan con http:// o https:// no cambian.
  • Las URL absolutas que comienzan por / (barra) probablemente se relativizan. En la mayoría de los casos, cambiar las direcciones URL que apunta a los archivos que residen en un directorio de puntos de vista. En estas situaciones, se puede simplemente eliminar la inicial /.Por ejemplo, el cambio / assets / css / main.css a los activos / css / main.css.
