A continuación se describe brevemente la estructura de directorio para los archivos de código fuente.

Los archivos específicos de su curso.

Debe modificar los siguientes archivos (excepto archivos lib) para poner en práctica el curso:

  • - assets: distintos archivos que componen el contenido de su curso
    • css - hoja de estilos en cascada (CSS) archivos
    • img - imágenes
    • js - actividad y la evaluación de los archivos JavaScript
    • lib - Bibliotecas
    • puedes añadir aquí subcarpetas adicionales para su contenido (PDF, ...)
  • - data: archivosCSV de datos que definen la secuencia de unidades y lecciones del curso
  • course.yaml - archivo principal de configuración

 

Archivos comunes para todos los cursos.

Usted debe evitar modificar estos archivos para simplificar la migración a versiones futuras del desarrollador del curso:

  • - controllers controladores de solicitudes web
  • - models clases comunes de infraestructura y entidades de base de datos
  • - modules de diferentes módulos de extensión
    • admin - consola de administrador
    • announcements - Anuncios
    • dashboard  panel de control
    • i18n - archivos de traducción para apoyar la interfaz de usuario en varios idiomas
    • oeditor - editor genérico AJAX utilizada por otros módulos
  • lib - bibliotecas de terceros
  • tests - todas las pruebas unitarias y pruebas funcionales
  • tools - estructura del curso y actividades / herramientas de verificación de evaluación
  • views - plantillas HTML de página para las diversas páginas del curso
