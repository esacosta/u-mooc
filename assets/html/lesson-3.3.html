Ya ha descargado la aplicación de ejemplo y ha conseguido ejecutar el servidor de desarrollo. Antes de empezar a modificar la aplicación para su propio curso, echemos un vistazo a las distintas páginas que un estudiante puede ver.

Explore el ejemplo de código. Esta página le muestra cómo los archivos que tocar para crear los resultados necesarios.

Las diferentes páginas que un estudiante puede ver son las siguientes:

    • Antes de la inscripción
    • Registro
    • Curso página de inicio
    • Una lección
    • Una actividad
    • Una evaluación
    • El foro de discusión
    • Anuncios
    • Perfil del estudiante

Antes de la inscripción

Cuando los futuros estudiantes visitan por primera vez el curso, la aplicación muestra una versión de la página principal del curso. En esta versión, el programa del curso es visible, pero los estudiantes no pueden acceder al contenido. La página también muestra un botón Register.

Esta página consta de tres áreas:

  • La cabecera que contiene
    • Icono del curso y el nombre de tu curso.
    • Barra de navegación horizontal, con dos pestañas de la izquierda (Curso y Registro) y una zona con un enlace de conexión. Observe que la barra de navegación horizontal es diferente una vez que el estudiante complete el registro.
  • El área de contenido contiene los nombres de cada unidad y la evaluación de cada uno. Ninguno de estos datos se puede hacer clic antes de la inscripción.
  • El pie de página contiene los mismos dos enlaces como para las páginas de registro.

La aplicación utiliza cuatro archivos para esta página:

  • views / base_registration.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a una persona no registrada.
  • views / preview.html define la estructura del área de contenido. Esta página es muy similar al registrado, vistas / course.html.
  • views / summary.html proporciona la descripción del curso y material introductorio. Esta página también es utilizado por views / course.html.
  • data / unit.csv proporciona el contenido real del área de contenido. Esta página también es utilizado por views / course.html.

Es necesario cambiar estos archivos para personalizar la aplicación para su material.

Registro

Cuando los futuros estudiantes visitan la página de registro, ven;

Esta página consta de tres áreas:

  • La cabecera contiene
    • Curso icono y el nombre de tu curso.
    • Barra de navegación horizontal, con dos pestañas de la izquierda (Curso y Registro) y una zona de dirección de la persona y un enlace Logout.
  • El área de contenido incluye un formulario para el futuro estudiante de llenar.
  • El pie de página contiene dos enlaces - uno a su sitio web y una de las condiciones de privacidad y términos para su curso.

La aplicación utiliza dos archivos para que esta página:

  • views / base_registration.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a una persona no registrada.
  • views / register.html define el contenido de la ficha de inscripción.

Una vez que un futuro estudiante envía el formulario de inscripción, la aplicación muestra la página de confirmación de registro:

Esta página consta de las mismas tres áreas como la página anterior y utiliza estos archivos para mostrar la página:

  • views / base_registration.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a una persona no registrada.
  • views / confirmation.html define el contenido de la ficha de inscripción.

Es necesario cambiar estos archivos para personalizar la aplicación para su material.

Curso página de inicio

Después de registrarse correctamente

Esta página consta de tres áreas:

  • La cabecera contiene
    • Curso icono y el nombre de tu curso.
    • Horizontal barra de navegación, con varias pestañas a la izquierda (Anuncios, Campo, Foro y mi perfil) y un área de dirección de la persona y un enlace Logout. Observe que la barra de navegación horizontal es diferente una vez que el estudiante complete el registro.
  • El área de contenido contiene enlaces a las unidades y de cada evaluación, así como información acerca de la disponibilidad de cada uno de esos enlaces.
  • El pie de página contiene los mismos dos enlaces como para las páginas de registro.

La aplicación utiliza cuatro archivos para que esta página:

  • views / base.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • views / course.html define la estructura del área de contenido. Esta página es muy similar a su contraparte no registrado, views / preview.html.
  • views / summary.html proporciona la descripción del curso y material introductorio otro. Esta página también es utilizada por views / preview.html.
  • data / unit.csv proporciona el contenido real del área de contenido. Esta página también es utilizado por views / preview.html.

Es necesario cambiar estos archivos para personalizar la aplicación para su material. 

Una lección

Cuando los estudiantes clic en una unidad desde la página principal, se ve a la página de la primera lección de la unidad que eligió. Desde allí, se puede ir a cualquier otra lección o actividad en la unidad:

Esta página se compone de cuatro áreas:

  • La cabecera contiene
    • Icono del curso y el nombre de tu curso.
    • Barra de navegación horizontal, con dos pestañas de la izquierda (Curso y Foro) y un área de dirección de la persona y un enlace Logout.
    • Una ruta de navegación
      Curso > Unidad 1> Lección 1.
    • El nombre y número de la unidad.
  • Una barra de navegación vertical, que consiste en una lista de todas las clases y actividades de la unidad actual.
  • El área de contenido es más compleja que las páginas anteriores. Contiene:
    • El nombre de esta lección.
    • Un botón para acceder a una versión de texto de la lección.
    • Una lista de los objetivos de la lección.
    • El vídeo lección.
    • Botones Atrás y Siguiente para navegar a través de la unidad. (La primera lección de la unidad no incluye el botón Atrás.)
  • El pie de página contiene los mismos dos enlaces como para las páginas de registro.

La aplicación utiliza tres archivos para que esta página:

  • vistas / base.html define las dos primeras partes de la cabecera y el pie de página define. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • vistas / unit.html define las segundas dos líneas de la cabecera, la navegación vertical, y el área de contenido.
  • data / lesson.csv contiene el contenido del área de contenido para todas las clases en el curso.

Es necesario cambiar estos archivos para personalizar la aplicación para su material.

Una actividad

Cada unidad puede tener un cierto número de actividades. Una página de actividad comparte casi la misma estructura que una página de la lección:

La única diferencia entre una página de la lección y una página de actividades (al menos en apariencia) es que en vez de un vídeo para el contenido principal, una actividad tiene un resumen de los puntos importantes de la lección anterior y tiene un conjunto de preguntas, con lugares para contestar las preguntas y revisar sus respuestas u obtener pistas.

La aplicación utiliza cuatro archivos para que esta página:

  • views / base.html es la misma que para las clases. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • views / activity.html define las mismas partes que hace views / unit.html para las lecciones. Es posible que desee ser coherente en estos archivos.
  • assets/js/activity- NM. js contiene una matriz de JavaScript que llena en la pregunta actual y las áreas de respuesta para la actividad. Cada sección de actividad tiene su propio archivo.

Es necesario cambiar estos archivos para personalizar la aplicación para su material.

Una evaluación

En ciertas partes de todo el curso, los estudiantes toman las evaluaciones para ver cómo lo están haciendo. 

Esta página consta de tres áreas:

  • La cabecera es la misma que para la página de inicio.
  • El área de contenido contiene las preguntas y las áreas de respuesta y un botón submit.
  • El pie de página es la misma que para la página de inicio.

La aplicación utiliza tres archivos para que esta página:

  • views / base.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • views / assessment.html define la estructura del área de contenido.
  • assets/js/assessment- {{sufijo}}}. js contiene una matriz de JavaScript que llena en la pregunta actual y las áreas de respuesta para la evaluación. Cada evaluación tiene su propio archivo.

Es necesario cambiar estos archivos para personalizar la aplicación para su material.

El foro de discusión

En cualquier momento, el estudiante puede decidir visitar el foro de discusión del curso

Esta página consta de tres áreas:

  • La cabecera es la misma que para la página de inicio.
  • El área de contenido incluye el foro de discusión.
  • El pie de página es la misma que para la página de inicio.

La aplicación utiliza dos archivos para que esta página:

  • views / base.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • views / forum.html define la estructura del área de contenido. También tiene un enlace al foro de discusión. Es necesario cambiar el vínculo que se crea para probar el foro.

Anuncios

En cualquier momento, el estudiante puede decidir visitar una página que contiene anuncios para el curso

Esta página consta de tres áreas:

  • La cabecera es la misma que para la página de inicio.
  • El área de contenido incluye los anuncios.
  • El pie de página es la misma que para la página de inicio.

La aplicación utiliza dos archivos para que esta página:

  • views / base.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • vistas / announcements.html define la estructura de la zona de contenido y contiene los anuncios reales.

Es necesario cambiar announcements.html a ser relevantes para su curso.

Perfil del estudiante

Por último, en cualquier momento, los estudiantes pueden visitar una página de perfil que muestra información sobre su situación actual en el curso y también les permite hacer algunos cambios.

Esta página consta de tres áreas:

  • La cabecera es la misma que para la página de inicio.
  • El área de contenido incluye el perfil del estudiante.
  • El pie de página es la misma que para la página de inicio.

La aplicación utiliza dos archivos para que esta página:

  • views / base.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a un estudiante registrado.
  • views / student_profile.html define la estructura del área de contenido.

Además, la aplicación obtiene y establece la información sobre el estudiante a partir del objeto del Estudiante.

Una de las acciones que los estudiantes pueden tomar en esta página es anular la inscripción en el curso.

La aplicación utiliza dos archivos para hacer que cada una de estas páginas:

  • views / base_registration.html define el encabezado y pie de página. Esta página es utilizada por todas las páginas que se muestran a una persona no registrada.
  • views / unenroll_confirmation_check.html define el contenido de la página que le pide confirmación antes de haber anulado el registro
    views / unenroll_confirmation.html     define el contenido de la página que confirma el estudiante es realmente anulado el registro.
