Supongamos que tenemos esta clase nativa en C++

// native class

 

class Student

{

private:

  char *_fullname;

  double _gpa;

public:

  Student(char *name, double gpa)

  {

    _fullname = new char [ strlen(name+1) ];

    strcpy(_fullname, name);

    _gpa = gpa;

  }

  ~Student()

  {

    delete [] _fullname;

  }

  double getGpa()

  {

    return _gpa;

  }

  char *getName()

  {

    return _fullname;

  }

};

Ahora, para envolver, seguimos esta pauta simple:

1.       Crear la clase administrada y que tiene una variable de miembro apuntando a la clase nativa.

2.       En el constructor o en otro lugar que creas adecuada, construir la clase nativa en el montón nativo (usando "nuevo").

3.       Pasar los argumentos al constructor según sea necesario; algunos tipos de que necesita a medida que pasan lograron no administrado.

4.       Crear recibos para todas las funciones que desea exponer en su clase administrada.

Asegúrese de que eliminar el puntero nativo en el destructor de la clase administrada.

Aquí está nuestro contenedor administrado para la clase de estudiante:

// Managed class

 

ref class StudentWrapper

{

private:

  Student *_stu;

public:

  StudentWrapper(String ^fullname, double gpa)

  {

    _stu = new Student((char *)

           System::Runtime::InteropServices::Marshal::StringToHGlobalAnsi(

           fullname).ToPointer(),

      gpa);

  }

  ~StudentWrapper()

  {

    delete _stu;

    _stu = 0;

  }

 

  property String ^Name

  {

    String ^get()

    {

      return gcnew String(_stu->getName());

    }

  }

  property double Gpa

  {

    double get()

    {

      return _stu->getGpa();

    }

  }

};

 

Envolver devoluciones de llamada de C

 

En esta sección, le mostraremos cómo tener las devoluciones de llamada C llamar .NET. Para fines de Ilustración, envolvemos la API EnumWindows(). Este es el esquema del código a continuación:

 

1.       Crear una clase administrada con delegados o una función que se llamará cuando se alcanza la devolución de llamada nativa.

2.       Crear una clase nativa que tiene una referencia a nuestra clase administrada. Podemos lograr esto mediante el uso de la gcroot_auto de la cabecera de la vcclr.h.

3.       Crear el procedimiento de devolución de llamada de C nativo y pasarlo como un parámetro de contexto (en este caso, el lParam) un puntero a la clase nativa.

4.       Ahora, dentro de la devolución de llamada nativa y con el contexto que la clase nativa, podemos obtener la referencia a la clase administrada y llame al método deseado.

A continuación se proporciona un ejemplo corto:

using namespace System;

 

#include <vcclr.h>

 

 

// Managed class with the desired delegate

 

public ref class MyClass

{

public:

  delegate bool delOnEnum(int h);

  event delOnEnum ^OnEnum;

 

  bool handler(int h)

  {

    System::Console::WriteLine("Found a new window {0}", h);

    return true;

  }

 

  MyClass()

  {

    OnEnum = gcnew delOnEnum(this, &MyClass::handler);

  }

};

La clase nativa creada con el propósito de sostener una referencia a nuestra clase administrada y por acoger el procedimiento de devolución de llamada nativa:

class EnumWindowsProcThunk

{

private:

  // hold reference to the managed class

 

  msclr::auto_gcroot<MyClass^> m_clr;

public:

 

  // the native callback

 

    static BOOL CALLBACK fwd(

    HWND hwnd,

    LPARAM lParam)

  {

      // cast the lParam into the Thunk (native) class,

      // then get is managed class reference,

      // finally call the managed delegate

 

      return static_cast<EnumWindowsProcThunk *>(

            (void *)lParam)->m_clr->OnEnum((int)hwnd) ? TRUE : FALSE;

  }

 

    // Constructor of native class that takes a reference to the managed class

 

  EnumWindowsProcThunk(MyClass ^clr)

  {

    m_clr = clr;

  }

};

 

Y ahora poniendo todo junto:

int main(array<System::String ^> ^args)

{

  // our native class

 

  MyClass ^mc = gcnew MyClass();

 

    // create a thunk and link it to the managed class

 

  EnumWindowsProcThunk t(mc);

 

    // Call Window's EnumWindows() C API with the pointer

    // to the callback and our thunk as context parameter

 

  ::EnumWindows(&EnumWindowsProcThunk::fwd, (LPARAM)&t);

 

  return 0;

}

 

Al revés: De devoluciones de llamada a C

Ahora, este problema es aún más fácil, ya que podemos tener un puntero a una clase nativa dentro de la clase administrada. La solución puede ser descrita como:

1.       Crear una clase administrada con los delegados deseados que deben desencadenar la devolución de llamada nativa.

2.       Crear una clase administrada que enlazará entre la clase nativa (contiene la devolución de llamada) y la clase administrada anterior (el generador del evento).

3.       Crear una clase nativa con la devolución de llamada dado.

Con fines de demostración, creamos una clase "TickGenerator" logrado que se generará un evento OnTick cada rato, entonces una clase de INativeHandler (interfaz) que debe ser llamada por el classTickGeneratorThunk administrado. Una clase de MyNativeHandler es una implementación simple de la INativeHandler que le muestre cómo establecer su propio controlador.

public delegate void delOnTick(int tickCount);
 
ref class TickGenerator
{
private:
  System::Threading::Thread ^_tickThread;
  int _tickCounts;
  int _tickFrequency;
  bool _bStop;
 
  void ThreadProc()
  {
    while (!_bStop)
    {
      _tickCounts++;
      OnTick(_tickCounts);
      System::Threading::Thread::Sleep(_tickFrequency);
    }
  }
 
public:
  event delOnTick ^OnTick;
 
  TickGenerator()
  {
    _tickThread = nullptr;
  }
 
  void Start(int tickFrequency)
  {
    // already started
 
    if (_tickThread != nullptr)
      return;
 
    // p.s: no need to check if the event was set,
    // an unset event does nothing when raised!
 
    _tickCounts = 0;
    _bStop = false;
    _tickFrequency = tickFrequency;
 
    System::Threading::ThreadStart ^ts = 
      gcnew System::Threading::ThreadStart(this, &TickGenerator::ThreadProc);
    _tickThread = gcnew System::Threading::Thread(ts);
    _tickThread->Start();
  }
  
  ~TickGenerator()
  {
    Stop();
  }
 
  void Stop()
  {
    // not started?
 
    if (_tickThread == nullptr)
      return;
    _bStop = true;
 
    _tickThread->Join();
    _tickThread = nullptr;
  }
};

Ahora la interfaz del controlador no administrado

#pragma unmanaged
// Create a simple native interface for handling ticks
 
// Native classes implement this class to add custom OnTick handlers
 
class INativeOnTickHandler
{
public:
  virtual void OnTick(int tickCount) = 0;
};

Una implementación simple

class MyNativeHandler: public INativeOnTickHandler
{
public:
  virtual void OnTick(int tickCount)
  {
    printf("MyNativeHandler: called with %d\n", tickCount);
  }
};


Ahora, a logrado crear el puente entre administrado y no administrado:

#pragma managed
// Create the managed thunk for binding between the native
// tick handler and the tick generator managed class
 
ref class TickGeneratorThunk
{
private:
  INativeOnTickHandler *_handler;
public:
  TickGeneratorThunk(INativeOnTickHandler *handler)
  {
    _handler = handler;
  }
 
  void OnTick(int tickCount)
  {
    _handler->OnTick(tickCount);
  }
};

Si lo ponemos todo junto

int main(array<System::String ^> ^args)
{
  // Initiate the native handler
 
  MyNativeHandler NativeHandler;
 
  // Create the tick generator class
 
  TickGenerator ^tg = gcnew TickGenerator();
 
  // Create the thunk and bind it with our native handler
 
  TickGeneratorThunk ^thunk = gcnew TickGeneratorThunk(&NativeHandler);
 
  // Bind the ontick event with the thunk's onclick event
 
  tg->OnTick += gcnew delOnTick(thunk, &TickGeneratorThunk::OnTick);
 
  // Start the tick generator
 
  tg->Start(1000);
 
  // Wait for user input
 
  Console::ReadLine();
 
  // Stop the generator
 
  tg->Stop();
 
  return 0;
}
