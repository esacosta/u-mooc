Esta página describe los datos de evaluación estudiantil y almacenado por supuesto desarrollador del curso y el proceso para exportar sus datos en archivos de texto en CSV formato. Vamos a empezar con los pasos para exportar los datos, seguido de una descripción de los datos disponibles.

    • Exportaciones proceso

Antes de empezar, asegúrese de que ha descargado e instalado el Google App Engine SDK para Python y que tiene Python 2.7 en su sistema. Para obtener información sobre cómo descargar e instalar el Google App Engine SDK, vea Desarrollar con Google App Engine .

Exportaciones proceso

El curso Builder aplicación almacena datos en Google App Engine almacén de datos de replicación de alta. Usted descarga los datos del curso utilizando el Google App Engine mayor herramienta de carga, que se incluye con el Google App Engine SDK de Python para la plataforma. Los datos son descargados utilizando el formato CSV, lo que hace que sea fácil de cargar los datos en una hoja de cálculo o en Fusion Tables para la visualización y análisis.

Puede exportar datos, ya sea desde el servidor de desarrollo o desde el servidor de producción. Los pasos son los mismos. La única diferencia es que al descargar datos desde el servidor de producción, es necesario utilizar su aplicación específica de contraseña, como se describe enConfiguración de una contraseña específica de aplicación.

Para exportar los datos del curso Builder, primero decida en el directorio en el cual poner los archivos exportados. Copie el bulkloader.yaml a ese directorio. Asegúrese de que el directorio no contiene tanto appengine_config.py y bulkloader.yaml. De lo contrario, usted no será capaz de exportar los datos.

Ahora, abra una línea de comandos o el comando. En esa ventana:

AppPath cd / datos 
cp .. / bulkloader.yaml. 
appcfg.py download_data - archivo_config = bulkloader.yaml - filename = DataFilename - tipo EntityType = - url = URL / remote_api

donde:

AppPath

El directorio que contiene el código de su aplicación.

DataFilename

Ruta de acceso a los datos exportados presentar este comando crea.

EntityType

¿Qué tipo de entidad a la exportación. Uno de lección, los estudiantes, o Unidad.

URL

La dirección URL para el servidor. 
Para su servidor de desarrollo, esto es http://localhost: puerto, donde puerto es el puerto que ejecuta el servidor. 
Para el servidor de producción, esto se APP_ID. Appspot.com, donde APP_ID es el nombre de la aplicación que registra.

Por ejemplo, suponga que mis archivos del curso vivir en ~ / mycourse y que desea exportar datos de los estudiantes de mi servidor de desarrollo en un archivo denominado student.csv en el directorio actual. Me gustaría utilizar los siguientes comandos:

cd ~ / mycourse / datos 
cp .. / bulkloader.yaml. 
appcfg.py download_data - archivo_config = bulkloader.yaml - filename = student.csv - clase Student = - = url http://localhost:8080/remote_api

El comando appcfg.py genera los siguientes archivos:

Expediente

Descripción

bulkloader-log-timestamp

Un archivo de registro que muestra los pasos del proceso. Utilice la opción - skip db_filename = indicador para omitir la generación de este archivo.

bulkloader-progreso-timestamp. SQL3

Una base de datos SQL3 archivo que muestra el progreso. Utilice la opción - skip db_filename = indicador para omitir la generación de este archivo.

bulkloader-resultados-timestamp. SQL3

Los datos exportados del curso en formato SQL3. Utilice la opción - skip db_filename = indicador para omitir la generación de este archivo.

nombre de archivo. csv

Los datos exportados del curso en formato CSV.

Ahora puede importar archivos CSV en una hoja de cálculo o en Fusion Tables para la visualización y análisis. Para ayudar a la importación de datos en tablas dinámicas, vea Importar datos.

Ahorro de tiempo y costes: exportación subconjuntos de datos

Por defecto, al descargar los datos del campo de Builder, Descargas de Google App Engine todos los campos para el tipo de entidad que se especifique. Para las lecciones y unidades, esto es probablemente una cantidad bastante pequeña de datos. Además, para aquellas entidades, la cantidad de datos no cambia en función del tamaño curso.

Entidades estudiantes, sin embargo, son diferentes. Si el curso es relativamente pequeña, la descarga de todos los datos de ese curso es bastante rápido y es gratis. Pero si el curso tiene un número muy grande de estudiantes, el comando appcfg.py puede tardar horas en ejecutarse y pueden incurrir en costos Google App Engine . Si usted encuentra que la descarga de datos toma mucho tiempo, considere descargar sólo el subconjunto de datos que realmente necesita.

En el comando appcfg.py anteriormente, uno de los parámetros es -.. Archivo_config = / bulkloader.yaml. Este parámetro le dice a Google App Engine para utilizar la información en .. / bulkloader.yaml para determinar qué descargar para cada tipo de entidad. Si desea descargar un subconjunto de los datos, cree un nuevo archivo bulkloader.yaml en un directorio diferente (como el directorio de la salida) y ejecute appcfg.py señala a ese archivo.

Por ejemplo, supongamos que los archivos del curso vivir en ~ / mycourse y que desea descargar a su directorio sólo las direcciones de correo electrónico de los estudiantes matriculados. Haga lo siguiente:

1. Haga una copia de AppPath / bulkloader.yaml como ~ / bulkloader.yaml, donde AppPath es el directorio que contiene el código de su aplicación.

2. Edite ~ / bulkloader.yaml.

3. Cambie la sección que comienza - tipo: Student para eliminar la información sobre todas las demás propiedades de la dirección de correo electrónico. Esto es:

  - Tipo: Estudiante
    Conector: csv
    property_map:
      - Propiedad: __ key__  
        external_name: email
        export_transform: datastore.Key.name

4. Ejecute los siguientes comandos:

  cd ~ / mycourse / datos
  appcfg.py download_data - archivo_config = ~ / bulkloader.yaml - filename = ~ / student.csv - clase Student = - = url http://localhost:8080/remote_api

Para el análisis de datos, puede optar por descargar otros subconjuntos de los datos de los estudiantes. Para ello, cambie ~ / bulkloader.yaml para eliminar sólo la información que usted no desea.

Los tipos de datos disponibles

En las tablas siguientes se describen los almacenes de datos Curso Builder para su curso. Tiendas Curso Builder varios tipos de datos, conjuntos más importantes son: Estudiantes, StudentAnswersEntity y EventEntity.

Tabla de alumnos contiene información acerca de los estudiantes, su estado de matrícula y los resultados de la evaluación.StudentAnswersEntity tabla contiene las respuestas detalladas más recientes proporcionados por un estudiante por cada evaluación.EventEntity tabla contiene los eventos a nivel de sistema y datos históricos sobre cada actividad o evaluación completa. Registro de eventos está desactivada por defecto y se puede activar en Ajustes> Admin.
